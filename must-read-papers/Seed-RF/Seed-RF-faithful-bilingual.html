**摘要**  
统一多模态模型（Unified multimodal models, UMMs）旨在在单个模型中同时处理感知和生成。  
简要解释：这一句说明 UMMs 的目标是把“理解”和“生成”统一到一个模型里。

然而，现有的 UMMs 在图像生成中仍然依赖一个冻结的、单独预训练的 VAE，从而引入了一个结构性瓶颈。  
简要解释：这里指出当前方法虽然“统一”了框架，但图像生成部分仍要借助外部 VAE，限制了端到端能力。

如果天真地移除它，就会引入质量差距，因为模型必须直接从原始像素中同时学习高层结构和低层细节。  
简要解释：直接去掉 VAE 并不一定更好，因为模型会一下子承担过多难度，导致生成质量下降。

在本文中，我们提出 Representation Forcing（RF），一种通过让表示预测成为模型原生能力来弥合这一差距的技术。  
简要解释：作者的核心想法是，不把“预测表示”当作外部模块，而是让模型自己学会这一中间表示生成能力。

具体而言，RF 强制解码器以自回归方式在像素之前预测视觉表示，作为中间 token；随后这些 token 会保留在上下文中，以在同一 backbone 内引导像素 diffusion。  
简要解释：RF 先生成“表示 token”，再用这些 token 带着像素扩散过程走，相当于在同一模型内部建立一个更顺滑的生成路径。

通过把表示从感知输出转变为生成目标，RF 消除了对任何外部生成潜变量空间的需求。  
简要解释：这意味着表示不再只是“看图后的产物”，而是直接成为要生成的对象，因此不必依赖外部 latent space。

我们发现，RF 同时有利于理解和生成。  
简要解释：RF 不只是提升生成，也会帮助模型的理解能力。

在图像生成上，我们带有 RF 的像素空间模型匹配了最先进的基于 VAE 的统一模型。  
简要解释：作者强调，去掉 VAE 后只用像素空间训练，性能仍能追平当前 SOTA。

在图像理解上，像素空间 RF 通常优于其基于 VAE 的变体。  
简要解释：在理解任务里，直接在像素空间做 RF 的效果一般还更好。

综合来看，这些结果为走向端到端、无瓶颈的 UMMs 提供了一个有效步骤。  
简要解释：本文声称 RF 是把统一多模态模型推进到“真正端到端”的关键进展。

一种自然的替代方案是直接在像素空间中生成。近期工作已经表明，像素空间扩散对于独立的生成模型是可行的 [8, 29, 47]。然而，我们发现，将这些方法直接应用到 UMMs 中，无法达到基于 VAE 的对应方法的质量。我们认为，这是由于 UMMs 具有更广泛的图像分布和更丰富的文本条件：模型必须从同一原始信号中同时学习图像的高层语义结构和细粒度细节。这促使我们引入一种中间表征，将这两个因素分离开来，使扩散过程可以专注于低层渲染，而不必退回到外部潜在空间。

简要解释：这一段解释了为什么“直接像素生成”在统一多模态场景里不够好，因为模型同时要处理语义结构和细节纹理，难度比单纯生成模型更高。

关键问题在于，这样的表征应当来自哪里。我们观察到，UMMs 内部其实已经提供了一种表征：它们的理解路径会学习视觉表征，用来捕捉高层结构，例如物体身份、空间布局和场景组成。在理解任务中，编码器 [37, 43, 45, 61] 从观察到的图像中提取这些表征。而在生成任务中，由于没有图像可用，模型必须仅从输入上下文中生成这些表征。这意味着模型必须学会自主预测这些表征。

简要解释：这一段指出，模型并不缺少“表征”这种中间表示，真正的问题是生成时如何从文本上下文把这些表征预测出来。

在本文中，我们提出 Representation Forcing（RF），一种通过让模型具备原生的表征预测能力来弥合这一差距的方法。我们的核心思想是把高层视觉表征锚定到解码器自身中。具体来说，我们使用理解编码器提取的视觉表征作为目标，并训练模型解码器在与语言相同的下一个词元预测目标下，以自回归方式预测这些表征。这些被预测出的表征在文本与像素之间提供了一个显式的结构支架；它们作为上下文条件保留在序列中，指导共享 Transformer 内的像素空间生成。通过把表征从感知输出转变为生成目标，RF 将理解与生成锚定在同一个表征空间中，而不依赖单独预训练的潜在空间（图1）。

简要解释：这一段给出了 RF 的核心做法，即先预测“视觉表征”再生成像素，让模型内部形成统一的中间表示，从而把理解和生成连接起来。

我们通过在线向量量化（online vector quantization）来完成这一离散化过程，它不需要单独预训练的 tokenizer。由于编码器是联合训练的，其特征会在整个训练过程中持续变化；因此，我们从编码器的指数滑动平均（EMA）副本中提取特征，以提供变化缓慢的目标，从而保持离散分配的稳定。具体来说，我们使用 EMA 编码器最后一层、最终 norm 之前的 patch 级特征。我们维护一个包含 `K` 个可学习原型嵌入（prototype embeddings）的 codebook；对于每个 patch 级特征，我们计算其与所有原型的余弦相似度，并将其分配给最近的原型，从而产生一个离散 token 索引。codebook 通过遵循 SwAV [4] 的 momentum update 进行在线更新，并且我们应用 Sinkhorn–Knopp balancing [28] 来防止 codebook collapse。这样，每个空间 patch 都会得到一个表示 token，形成一个与图像空间布局相对应的序列，并且可以按 raster-scan 顺序进行预测。

**简要解释：** 这一段在说明“表示 token”是怎么来的：不是直接回归连续向量，而是把图像 patch 的特征离散化成 token，这样更稳定，也更适合像语言模型一样自回归预测。

### 3.2

**通过预测表示生成像素**

定义好表示 token 之后，我们现在来描述解码器如何利用它们生成图像。在训练期间，EMA 编码器提供表示 token 作为真实目标（ground-truth targets），解码器学习在交叉熵损失下以自回归方式预测这些 token，并且它们与文本一起位于同一个 next-token prediction 流中。在推理时，编码器不再参与：解码器仅从文本提示中生成表示 token。我们把这个过程称为 Representation Forcing：一方面，理解编码器中的表示强迫解码器学习相同的高层视觉结构；另一方面，解码器预测出的表示强迫像素生成过程遵循预期的语义布局。

### 3.3 训练与推理

我们的模型遵循 BAGEL [11]，采用 Mixture-of-Transformers（MoT）架构。所有 token 共享相同的 multi-head self-attention 层，但会根据各自类型被路由到不同的模态专属 feed-forward experts 中。我们维护三组 experts：一组用于多模态理解，一组用于表示 token 预测，一组用于像素生成。每个表示 token 的嵌入由两个可学习嵌入之和构成：一个 2D 空间位置嵌入，以及一个按离散 token ID 索引的 token identity 嵌入。后者存储在一个独立的 \(K\)-entry 表中。

模型以端到端方式训练，目标函数为  
\[
L = L_{LM} + L_{FM} + L_{Rep},
\]
(3)

```python
score = matmul(Z, C.T) / t
score = softmax(score, dim=1)
score = softmax(score, dim=0)
A = argmax(score, dim=1)

# 通过 EMA encoder 提取连续特征
# 两两 cosine 相似度：(B*L, K)
# 按行归一化
# 按列归一化（Sinkhorn-Knopp，1 次迭代）
# 离散分配：(B*L,)

N_k, C_new = zeros(K), zeros(K, D)
A_c = A.view(B * L, 1).expand(B * L, K)
C_new = scatter_add(C_new, dim=0, index=A_c, src=Z)
N_k = scatter_add(N_k, dim=0, index=A, src=ones(B * L))
C_new = normalize(C_new / N_k, dim=1) # 根据分配得到的新原型
C = m * C + (1 - m) * C_new
# momentum 更新
C = normalize(C, dim=1)
```
