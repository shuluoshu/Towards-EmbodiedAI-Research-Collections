# WALL-WM：在事件关节处雕刻世界动作建模

X Square Robot Team（X Square Robot 团队）

WALL-WM 是一种 World Action Model（WAM，世界动作模型），它将视频-动作学习从以 chunk 为中心的优化，转向以事件为基础的 Vision-Language-Action（VLA）预训练，使用语义连贯的动作事件作为学习的原子单元。现有的 WAM 通常从多模态或视频基础模型初始化，然后在当前观测和指令条件下优化固定长度的 action chunks。虽然这样做方便，但这种以 chunk 为中心的表述造成了根本性的粒度不匹配。语言描述的是语义目标和事件，视觉通过连续的场景动态演化，而动作则在控制层时间尺度上运行；把三者强行塞进同一个固定长度的预测窗口，会把 VLA 训练变成短视野的相关性拟合。这不仅没有充分利用预训练得到的视觉-语义先验，还可能用 chunk 特定的动作捷径主动覆盖这些先验，削弱组合性和长时程泛化能力。WALL-WM 通过围绕语义事件来组织监督信号和数据，来解决这种不匹配。具体而言，它将事件级 caption 与 cluster-balanced sampling 构成的数据生态，与以事件为基础的 VLA 预训练配对，从而支持在多样行为、场景和任务结构上的可扩展学习。基于同一个经过事件预训练的 backbone，WALL-WM 支持两种互补的推理模式。event 模式接收下一事件描述，并支持可变长度的执行 chunk；而 unified 模式使用带有 Staircase Decoding 的 VLM，在保持梯度连续的 VLA 路径的同时，条件化传统的固定长度 chunk 推理。再结合基于 Muon-optimizer 的大规模预训练基础设施，WALL-WM 为通用型 WAM 提供了一套实用的规模化方案。实验表明，WALL-WM 在语言、场景和任务上都具有广泛的泛化能力，并在大规模真实世界泛化评测中取得了 state-of-the-art 的性能。

简要解释：这一段是在说明 WALL-WM 的核心思想是把学习单位从“固定长度动作块”改成“语义事件”，这样更贴合语言、视觉和动作各自不同的时间与语义尺度，也更利于长程泛化。

## 引言

近年来，具身基础模型（embodied foundation models）(97, 41, 9, 36, 8, 14, 17) 的进展，越来越多地由从多模态理解模型和视频基础模型继承而来的大规模先验所驱动（20, 37, 13, 95）。在本报告中，我们用 VLA 广义地指代那些从视觉-语言输入预测动作的具身基础模型，并用 WAM 指代那些显式地将未来观测建模与动作预测耦合起来的 world-action models（世界动作模型）。

简要解释：这一段先界定了本文中 VLA 和 WAM 的用法，并指出当前具身模型的能力主要来自多模态和视频基础模型的大规模预训练先验。

现有大多数具身基础模型，都是通过根据当前观测和语言指令来预测固定长度的动作块（fixed-length action chunks），从而对这些先验进行适配。这种以 chunk 为中心的表述是有效且方便的，但它掩盖了一个结构性不匹配：它用外部时钟切割具身动态，而语言、视觉和动作却是在不同的语义尺度和时间尺度上演化。语言指定目标和事件；视觉通过连续的场景动态演化；动作运行在控制级时间尺度上，并对接触、时机以及微小扰动非常敏感。因此，将预训练得到的视觉-语义先验适配到具身控制，并不只是一个微调问题。首先要回答的是：视频-动作学习的原子单元应该放在哪里。这个区别正是 WALL-WM 的起点：

其余论证都源于这种不匹配。核心挑战通常不是多模态融合本身，而是保持几何结构的对齐（geometry-preserving alignment）。文本、视觉和动作并不共享同一种邻域概念：语义上相似的指令可能诱发不同的视觉轨迹，而视觉上相似的状态可能需要不兼容的控制响应。如图 1 所示，文本提供粗粒度的语义对齐，视觉提供更稠密的时空落地，而动作则要求最精细的局部精度。一个有用的 WAM 必须把这些模态连接起来，而不能把它们压扁成一个无差别的嵌入空间。

简要解释：作者认为难点不只是把多种输入拼在一起，而是要保留它们各自的结构关系，避免不同模态在同一个空间里被“混平”。

视频是语言与动作之间最自然的支架（scaffold）。互联网规模（Internet-scale）的视频预训练捕获了丰富的视觉动态，否则这些动态只能通过具身交互来学习 (37, 20)。视频在语义上也足够结构化，能够在事件边界上与语言对齐；同时，它在时间上又足够稠密，能够暴露动作执行所需的时机、过渡与状态变化。未来视频与动作共享因果时间结构，这使得从视觉到动作的落地（visual-to-action grounding）以及逆动力学（inverse dynamics）成为可能 (20, 43)。从这个意义上说，视频提供了一条从高层语义意图到低层执行的、较少依赖具身经验的桥梁。

简要解释：视频既有语义结构，又有时间连续性，因此可以作为语言和动作之间的中介，帮助模型把“想做什么”连接到“怎么做”。

把这个支架转化为一个 WAM 并不是一个短暂的适配阶段。它是一种保留先验的提升（prior-preserving lift）：模型必须继承从大规模视频中学到的语义和时间结构，同时获得具身动作所需的可控性、接触敏感性和因果落地能力。简单地附加一个动作解码器，确实可以把动作接到视觉先验上，但它并不能决定这个先验应该在什么单位上变得可执行（executable）。如果没有这样的单位，联合优化就可能塌缩到数据最丰富的模态，或者用短时域的动作相关性覆盖掉有用的视觉-语义结构。

简要解释：作者强调，真正难的是“保留并升级”视频先验，而不是只在其上叠一个动作头；否则训练可能偏向数据最多的模态，破坏已有的高层语义结构。

这种提升带来两个要求。第一，WAM 必须保留视频先验：视频生成模型偏好语义不变性、视觉合理性和时间平滑性，而具身控制要求对动作诱发的分歧和接触过渡保持敏感 (86, 44)。第二，WAM 必须提供时间落地（temporal grounding）：语言指令通常描述全局任务或语义事件，而观测和动作……

简要解释：这里作者提出 WAM 的两个基本条件：一是不能丢掉视频模型已有的“像真、连贯、语义稳定”的能力，二是要把语言中的任务描述和具体时间上的观测、动作对应起来。

同样的推理也说明了：要从视频先验中提升（lift）到一个通用的 WAM（world-action model，世界-动作模型），必须包含什么：  
T1. 推理：把全局指令和任务进度转换为事件结构化的意图。  
T2. 视觉预测：保留从 caption 到 video 的归纳偏置，同时让未来观测能够被可执行事件所控制。  
T3. 精细操作：显式呈现动作执行所需的时序、接触转换以及局部状态变化。  
简要解释：这三点分别对应高层推理、视觉生成/预测和低层操作控制，强调 WAM 不能只做单一能力，而要把三层能力统一起来。

这些层次不能通过把一个 VLM、一个视频模型和一个动作头作为彼此独立的模块堆叠起来获得。它们需要一种训练范式，其中对齐单元同时对语言有意义、在视频中可见，并且能够通过动作执行。  
简要解释：作者强调，真正关键的不是模块堆叠，而是设计一个能同时对齐语言、视觉与动作的训练单位。

这给出了对齐单元的三个设计原则：  
• 几何保持（Geometry preservation）：在不把各自原生结构压缩为一个共享空间的前提下，连接语言、视频和动作。  
• 先验保持（Prior preservation）：与视频基础模型继承下来的 caption-to-video 结构保持兼容。  
• 可执行因果性（Executable causality）：提供一个具有清晰时间支撑的预测目标，同时允许持续时间跟随任务本身，而不是固定时钟。  
简要解释：这三条原则规定了理想的对齐单元应同时保留三种模态的结构、继承视频模型的先验，并且具备可执行、可控的时间因果关系。

这些原则排除了把固定长度动作块作为基本单元的做法。chunk 便于批处理和部署，但它并不是语言、视频和动作共同拥有的自然对象。它可能切到某个语义行为的中间，把多个行为合并成一个目标，或者甚至需要历史上下文来判断这个 chunk 到底应该表示什么。  
简要解释：这里说明固定长度 chunk 虽然工程上方便，但语义上不自然，容易破坏事件边界，导致表示混乱。

因此，WALL-WM 用一个以动作为基础的语义事件（action-grounded semantic event）来替代固定长度 chunk：也就是一个在时间上连贯的可执行行为片段，例如 reaching、grasping、lifting、moving 或 placing。它可以用语言表达，在视频中观察到，并通过动作实现。不同于遵循外部时钟的固定时间 chunk，这种以动作锚定的语义事件会在底层可执行行为发生变化时开始和结束。因此，它满足上述原则：语言命名该事件，视频锚定其空间-时间演化，动作通过控制将其实现。  
简要解释：这一段给出 WALL-WM 的核心单位定义。模型不再对齐“等长时间片”，而是对齐“语义事件”，即一个真正有动作意义的行为段。

WALL-WM 通过事件锚定的 WAM 预训练（event-grounded WAM pretraining）来实例化这一原则。事件 caption 与对应的视频和动作片段配对，模型被训练去在事件对齐的区间上对未来的视频和动作进行去噪。这不仅仅是把事件当作辅助条件；它是在事件层面上锚定训练问题本身。其结果是一条从视频基础模型走向可执行世界-动作模型的、保持先验的路径。  
简要解释：这里说明训练方式本身也按“事件”组织，而不是只把事件当额外输入，从而让模型学习到事件级别的未来预测能力。

在推理时，同一个经过事件预训练的 backbone 支持两种互补模式：  
Event mode. 系统在事件空间中展开。一个 VLM、人类或 agent 提出下一个事件描述，而 WALL-WM 在观察下一个状态之前，先执行与之对应的可变长度视频-动作片段。该模式遵循任务的自然持续时间，而不是固定的控制视野长度。  
Unified mode. 仍然保留传统的固定长度 chunk 推理，但该 chunk 不再仅仅以原始全局指令作为条件。带有 Staircase Decoding 的 VLM 提供关于任务进度的事件结构化潜在推理，生成一个潜在事件表示，用它来指导下一个局部 chunk，同时保持一条梯度连续的 VLA 路径。  
简要解释：这里给出两种推理方式。Event mode 直接按事件执行；Unified mode 仍按 chunk 运行，但通过事件级潜在推理来改进传统 VLA 流程。

2.2

生成式具身世界模型

生成式世界模型近年来已成为具身 AI 的一个有前景的范式，其中智能体学习预测物理世界未来状态，并利用这些预测进行规划和控制。早期基于模型的强化学习方法，如 PlaNet、Dreamer 和 DreamerV3（28, 27, 29），学习用于控制的紧凑预测状态；而 JEPA 系列，包括 V-JEPA 和 LeWorldModel（2, 54），表明在特征空间中进行预测可以在不显式重建像素的情况下捕捉有意义的时间结构和物理结构。更近期的具身世界模型进一步利用生成式视频和 world-action modeling 目标，从大规模异构数据中学习机器人-环境动力学。这些模型使用预测的未来帧或中间视觉表示作为计划（plans），再通过逆动力学模型（20）、中间特征动作解码器（59）、密集对应（dense correspondence）（43）、面向规划的轨迹解码（21, 94），或从人类视频和新场景中迁移而来的合成示范（6, 5, 15, 37, 51）恢复出可执行动作。近期工作进一步表明，视频生成器编码了有用的 3D 和交互先验（32, 42, 93, 78, 46），这促使人们将其用作具身动力学模型。沿着这一方向，LaDi-WM、AdaWorld、Motus、LDA-1B 和 MotuBrain（33, 23, 7, 53, 69）探索了潜在扩散（latent diffusion）、结构化视觉预测、潜在动作条件化以及统一的视频-动作建模，而 LingBot-VA（44）及相关的统一去噪架构（26, 22, 95, 13, 83, 76, 12, 88, 47）则联合建模未来预测和动作生成。更近期的方法，如 Fast-WAM（86），通过在测试时避免显式的视频解码或未来想象（future imagination），进一步提升了推理效率。总体而言，这些进展表明，显式的未来状态建模能够改善样本效率、鲁棒性和泛化能力。

简要解释：这一段说明“世界模型”为什么重要。它不只是输出动作，而是先预测未来会发生什么，再据此规划，因此通常更省样本、更加稳健，也更容易泛化。

2.3

潜在推理

另一条互补的研究路线是：将 chain-of-thought（CoT）推理导入紧凑的潜在表示中，而不是直接输出完整的文本推理轨迹，其动机在于推理效率，以及能够探索超越离散 token 序列的语义级推理轨迹（30, 24, 38, 39, 50, 92）。LaDiR（38）通过 VAE 将推理步骤编码为连续的“thought tokens”块，并用 latent diffusion model 对其进行细化；其强化学习扩展版本（39）进一步优化潜在轨迹，以保持解的多样性。近期 VLA 方法进一步将这一思想扩展到具身约束下的潜在推理：LaST0（49）提出了 latent spatio-temporal CoT，用于捕捉机器人操作中的未来视觉动态、3D 结构和本体感觉（proprioceptive）状态；LaST-VLA（52）将几何约束和 world-model 远见（foresight）蒸馏到用于自动驾驶的 latent spatio-temporal 表示中；LaRA-VLA（3）则将文本与视觉 CoT 监督逐步转移到潜在推理动力学中，以实现高效动作生成。相比之下，WALL-WM 使用 staircase parallel decoder，将潜在 CoT token 注入并沿着 VLM backbone 的错位深度（staggered depths）传播。这一设计在尽可能保留预训练 VLM 的层级视觉-语言先验和因果计算结构的同时，分摊了潜在推理的层级代价（layer cost）。

简要解释：这一段讲的是“把思维压缩到潜空间里”的思路。WALL-WM 的重点不是直接吐出长推理文本，而是把推理信息塞进模型内部的不同层里传播，这样更高效，也更能保留原始 VLM 学到的结构化先验。

**多视角视觉世界事件建模**

在该锚点上，动作塔（action tower）会在一个独立采样的噪声水平 \(t_A \sim \Phi_A\) 下向前传播每一个动作，同时对同一个锚点进行交叉注意力（cross-attends），因此动作塔可以在完整的动作调度（action schedule）上学习，而不会采样到不匹配的视频步。作为一种与推理无关、仅用于提升训练吞吐的技巧，我们还可以在每个优化器步中可选地抽取 \(K>1\) 个相互独立的动作噪声水平，同时复用同一次带锚定的视频前向传播；\(K\) 只是在计算上起到摊销（amortize）作用，在 rollout 时不起任何作用。符号化地表示为，u.a.r.
\[
t_k^A \sim \Phi_A,\quad k=1,\ldots,K,
\]
\[
m^{(j)}=s^\star \quad \text{for all } j \in \{0,\ldots,N_d-1\},
\tag{12}
\]
其中 \(\Phi_A\) 的支持集与密度均不受限制；在每个优化器步中，若启用该吞吐技巧，则代价为一次视频前向加上 \(K\) 次动作前向（否则 \(K=1\)）。

简要解释：这段在说明动作分支的训练方式：视频只前向一次，用它作为锚点，动作端可以在多个噪声层级上并行学习，从而提高训练效率，但不改变推理阶段的行为。

**动作目标（Action Objective）**。在动作预训练阶段，仅优化动作塔，并采用 flow matching 目标，而视频塔保持固定。我们的主实验使用 \(v\)-prediction，这与第 3.2 节中视频侧的目标对应。在上述非对称的 \(1\)-到-\(N_d\) 映射下，每次抽样 \(k\) 都使用独立采样的动作噪声水平 \(t_k^A \sim \Phi_A\)，同时复用同一次锚定的视频前向；主损失是这些抽样上的 flow matching 均方误差（MSE）的平均值，
\[
L_A=\frac{1}{K}\sum_{k=1}^{K} L_A^{k},\qquad
L_A^{k}=w(t_k^A)\,\hat{y}_A^{k}-y_A^{k\star},
\tag{13}
\]
其中 \(w(t_k^A)\) 可像第 3.2 节那样按噪声水平进行重新加权。默认情况下，\(\hat{y}_A^{k}\) 是预测速度 \(\hat{v}_A^{k}\)，目标为 \(v_A^\star=\epsilon_A-a_0\)。如果启用可选的 \(x\)-prediction，则网络改为输出干净动作（\(\hat{y}_A^{k}=\hat{a}_0\)，\(y_A^{k\star}=a_0\)）；上面的加权 MSE 形式不变。对于接触密集型数据（细粒度双臂操作、接触丰富的插入任务、恢复/重新抓取（re-grasp）片段），接触事件只跨越少数几帧。\(v\)-prediction 可能会让这些帧的权重过轻，尤其在高噪声下更明显，因此我们在这类实验中暴露 \(x\)-prediction 作为可选模式。我们还可以在恢复得到的轨迹 \(\hat{a}_0^{(k)}\) 上加入一个 Type-II DCT 辅助项，以强调整体运动形状而不是逐帧抖动（指数分箱衰减），并通过 \((1-t_k^A/T)^2\) 在高噪声下进行降权，其中 \(T\) 是动作扩散的最大时间步（噪声调度的末端）：
\[
L_A += \frac{w_{\mathrm{DCT}}}{K}\sum_{k=1}^{K} L_{\mathrm{DCT}}^{k},
\]
\[
L_{\mathrm{DCT}}^{k}=
\left(1-\frac{t_k^A}{T}\right)^2
\sum_{j=0}^{T_a^{-1}}
e^{-j/(T_a/4)}
\left(\mathrm{DCT}(\hat{a}_0^{(k)})_j-\mathrm{DCT}(a_0)_j\right)^2,
\tag{14}
\]
其中 \(\hat{a}_0^{(k)}\) 由第 \(k\) 次抽样恢复得到。我们的主实验使用 \(v\)-prediction，且 \(w_{\mathrm{DCT}}=0\)，因此除非另有说明，否则 \(x\)-prediction 和 DCT 辅助都处于关闭状态。

简要解释：这段定义了动作分支的训练损失。默认做法是预测速度场 \(v\)，必要时可切换到直接预测干净动作 \(x\)，而 DCT 辅助则是为了让模型更关注整体动作轮廓，尤其适用于接触稠密、短时关键事件很多的任务。

给定当前的多视角观测 \(\{V_0^{(v)}\}_v\) 和语言指令 \(\ell\)，VLM 产生隐藏状态 \(H_q\)。文本 token 通过边界掩码 \(m_{\text{txt}}\) 提取，并投影到 DiT 的条件空间：
\[
c_\ell = M_T\,P_Q\!\left(H_q[m_{\text{txt}}]\right),
\tag{15}
\]
其中 \(P_Q\) 表示一个可学习投影层，\(M_T\) 表示 DiT 主干中原生的文本 MLP。得到的条件序列会与图像 token 一起注入到 WAM 的交叉注意力层中。

简要解释：这段说明语言条件如何接入动作生成模型：先由 VLM 编码多视角图像和文本，再把文本表征映射到 DiT/WAM 能使用的条件空间，和图像 token 一起参与跨模态融合。

标准的串行 rollout 解码过程在长链 latent reasoning 上会出现严重的计算低效。传统的自回归 rollout 以串行方式逐个生成 latent reasoning states，这要求 Transformer 层进行穷尽式前向传播，并且在每个推理步骤重复计算重叠的视觉-语言特征。这种串行依赖瓶颈会显著增加推理延迟，限制具身操作任务的实时交互能力，并阻碍长时程推理 rollout 的可扩展性。为了解决传统 latent reasoning rollout 中的串行解码开销和冗余计算，我们提出一种新的 Staircase latent decoding 范式，以高效并行生成连续的 latent reasoning 序列。

简要解释：这段指出了长时程推理的主要瓶颈是串行解码太慢，因此提出 Staircase latent decoding，用并行化方式生成连续的潜在推理状态，从而降低延迟并提升实时性。

**阶梯式 latent CoT 解码。** 与其逐个自回归地生成推理 token，WALL-WM 将推理建模为一个由 𝐾𝑐 个连续潜在推理状态组成的紧凑序列：
\[
\hat{y}_{1:K_c}=\{\hat{y}_1,\ldots,\hat{y}_{K_c}\}.
\]
(16)

推理分支从微调后的 Qwen3.5-9B checkpoint 初始化，并实现为一种轻量级的 Mixture-of-Transformers (MoT) 结构，与冻结的 backbone 相耦合。为了实现并行的 latent rollout，我们在一个 relay depth 𝑁𝑟 处对 Transformer 进行分割，其中较低层编码共享的视觉-语言 grounding 特征，而较高层逐步专门化为不同的推理步骤。具体而言，只有第一个 latent position 经过较低的 Transformer 层，生成一个共享的 relay 表示，并在所有推理位置之间复用。随后，其余的 latent states 通过上层 Transformer blocks 并行生成，同时进行彼此独立的因果 cache 更新。这个阶梯式调度机制（staircase scheduling mechanism）将传统的顺序推理过程转换为一种深度并行的 latent rollout：
\[
\hat{y}_{1:K_c}=F_{\text{stair}}(x;N_r),
\]
(17)

其中 𝑁𝑟 控制将共享 grounding 计算与并行推理计算分隔开的 relay depth。与标准自回归解码相比，所提出的阶梯式设计避免了在每个推理步骤中重复计算底层的视觉-语言特征，从而显著降低了长时域具身 rollout 的推理延迟。由此得到的 latent reasoning states 是完全可微分的，并且无需离散 token 采样即可直接注入 WAM 的 cross-attention 通路中。

简要解释：这一段的核心是在说，WALL-WM 不再用传统“一个 token 接一个 token”的方式做推理，而是把推理压缩成一串连续潜变量，并通过分层并行计算来加速，同时保留可微分性，方便直接接入下游注意力模块。

**冻结的 latent-to-text reconstruction supervision。** WALL-WM 并不是直接蒸馏自回归 hidden states，而是通过一个冻结的 latent-to-text reconstruction objective (38, 92) 来监督 latent reasoning。生成的 latent reasoning states \(\hat{y}_{1:K_c}\) 先通过前缀投影器 \(P_{\text{pref}}\) 投影为 embedding space 中的 soft prefix \(z_{1:K_c}=P_{\text{pref}}(\hat{y}_{1:K_c})\)，该 embedding space 属于一个轻量级冻结语言模型（Qwen3.5-0.8B），后者以自回归方式重建对应的文本 CoT trace：
\[
P_{\phi}(r_{1:M_r}\mid z_{1:K_c}).
\]
(18)

训练目标定义为 token-level reconstruction loss：
\[
L_{\text{CoT}}=-\sum_{m=1}^{M_r}\log P_{\phi}\bigl(r_m\mid z_{1:K_c}, r_{<m}\bigr).
\]
(19)

只有阶梯式推理分支和前缀投影器 \(P_{\text{pref}}\) 会在训练中被优化，而 reconstruction language model 在整个训练过程中保持冻结。因此，latent reasoning states 被鼓励编码紧凑的高层推理语义，而不是复制精确的 token-level 解码轨迹。由此得到的 latent CoT 在具身 rollout 中充当一种可微分、紧凑且计算高效的显式文本推理替代方案。

简要解释：这一段说明训练时并不直接让 latent 状态去“模仿每个 token”，而是让它们通过一个冻结语言模型还能还原出原来的 CoT 文本，从而逼迫 latent 表示学到更抽象、更压缩的推理语义。

**推理模式。** 同一个经过 event-pretrained 的 backbone 通过这条 language-guided pathway 暴露出两种互补的 inference mode。在 event mode 中，系统在 event space 里展开：VLM、human 或 agent 会针对当前观测提出下一事件的描述，WALL-WM 则执行对应的可变长度 video–action segment，然后再观察下一个状态，因此 rollout 会遵循每个任务阶段的自然持续时间，而不是固定的控制时域。在 unified mode 中，阶梯式 decoder 会在一次并行前向中发出 𝐾𝑐 个连续 latent CoT states，并将它们精确插入到 WAM cross-attention 中原本由 atomic-instruction condition 所对应的位置。由于这两种模式都通过相同的 reasoning interface 来对 WAM 进行条件控制，WALL-WM 在单一模型中统一了开放式、事件驱动的 rollout，以及标准的固定时域 VLA 执行。

简要解释：这一段在讲同一个模型可以切换两种工作方式：一种按“事件”推进，适合开放式长任务；另一种按固定长度一次性输出 latent CoT，适合标准 VLA 执行。两者共享同一套推理接口，因此模型结构和使用方式被统一起来。

4.3 事件中心的层级式 Captioning（Event-Centric Hierarchical Captioning）

图 8 中 action-paired sources 里的每个 episode 都配有一个时间上嵌套的 caption 层级，其时间跨度锚定在原子操作动作（66）上——例如 reach、grasp、close、lift 和 place 这类短小、可执行的原语——而不是固定的帧窗口。具体来说，我们先在动作边界处切分同步的视频-动作流，然后再分配 captions，使每个语言区间都与视觉和控制中的同一物理区间对齐。我们并不把单一的自由形式描述分配给整条轨迹，而是把每个 episode 分解为四个核心时间抽象层级，再加上一个可选的人类标注层。这样设计是为了同时保留示范的全局意图，以及决定机器人如何真正完成任务的局部事件结构。四个核心层级分别是 Task、Subtask、Action 和 Segment。Task 层是覆盖整个 episode 的字符串，用来概括总体目标，例如打开抽屉、把物体放入容器，或擦拭表面。Subtask 层把 episode 划分为少量连续、语义上有意义的阶段。这些阶段通常对应中间目标，例如接近目标物体、建立抓取、搬运物体，以及放置或释放它。Action 层进一步把每个 subtask 细化为更短的 manipulation primitives，例如 reaching、对齐夹爪、闭合手指、抬起、平移、插入或回撤。最后，Segment 层提供最细粒度的时间分解，捕捉可能仅持续整条轨迹极小一部分的短暂局部事件。第五个 Human 层保存人工标注的片段。该层只为收到人工标注的 episode 子集填充，并发挥两个互补作用。第一，它为验证自动生成的层级提供高质量监督。第二，它为我们提供一个可信参考，用于评估 caption 准确性、时间边界质量，以及跨任务语义标签的一致性。实际上，Human 层并不是每个 episode 都必需；核心的四层层级使数据集能够扩展，而人工子集则为标注流程提供锚点并支持质量控制。

**4.4 视觉-语言与动作上的聚类平衡采样（Cluster-Balanced Sampling over Vision-Language and Action）**

production recipe 仅在我们大规模自收集的 embodied corpus 的一个小子集上进行训练，这个子集由下述 cluster-balanced sampler 选出。这个下采样并不只是预算上的妥协：它源于这样一个观察，即原始 embodied corpus 呈现出高度长尾分布，而第 4.3 节中的 multi-level caption schema 揭示了关于 vision、language 和 action 的细粒度统计。尤其是，caption windows 在聚类之前会按照 action boundaries 进行分段，而不是按均匀的时间切片来划分。这个 action-aligned decomposition 会把每个长 demonstration 转化为语义上局部化的单元，使得最终得到的 language 和 vision-language 分布比原始的 task-level mixture 更加平衡。  
简要解释：这里强调，先按动作边界切分再聚类，能把长视频拆成更有语义的局部单元，从而缓解长尾分布问题。

在这个 multi-level schema 之上，WALL-WM 运行两个离线聚类（offline clustering）流程，其目的是为 dataloader 提供一个可控的采样分布。第一种是 vision-language（VL）聚类：一个冻结的 multimodal encoder 将每个视觉观测与 caption 对映射到联合 embedding 中，随后聚类过程把这个空间划分为 topic clusters，用以概括语料库在 instruction-scene 覆盖上的分布。第二种是 action 聚类：action chunks 在 trajectory space 中分别进行聚类，其中长尾部分集中着非标称（non-nominal）运动，例如 recoveries、re-grasps、retries，以及由接触驱动的 corrections。这些恰恰就是 action-aligned multi-level schema 旨在显露出来的行为。重要的是，在从 task-level captions 经过基于 action 的分解，进一步变为更细粒度的 action 和 segment captions 之后，language-only 和 vision-language 两种聚类视角都会变得更加均匀，被少数主导性 topic 吸收的样本更少。这为平衡采样提供了一个实际基础：稀有但重要的 instruction-scene 模式和 trajectory 模式，成为显式的采样单元，而不再被高频动词和标称运动所淹没。  
简要解释：这一段说明 WALL-WM 用两套离线聚类来控制采样分布，一套看视觉-语言主题，一套看动作轨迹，从而把稀有但关键的恢复动作单独凸显出来。

简要解释：这里说明平衡采样在训练中的实际作用是让模型既覆盖更多指令-场景组合，又能学会“同一动词、不同动作”的细微差别，而且几乎不增加训练开销。

**4.5 通过接触丰富的随机初始化恢复数据**

预训练通过加入恢复 / 接管（recovery / takeover）回合来增强，这些回合会显式覆盖接触丰富（contact-rich）事件周围的动作解空间。对于每个这样的事件 \(e\)，我们在末端执行器构型 \(q \in \mathbb{R}^{D_q}\) 上定义一个局部接触位姿分布 \(p(q \mid e)\)。它的支撑集位于以标称接触位姿 \(q^\star_e\) 为中心的小测地球 \(B_\epsilon(q^\star_e)\) 内，因此采样到的状态会保持在事件附近，同时保留其接触几何结构。

由此得到的预训练分布为
\[
\tilde{p}_{\text{train}}(q, e) = (1 - \alpha)\, p_{\text{nominal}}(q, e) + \alpha\, \mathbb{E}_e\!\left[p(q \mid e)\right],
\]
其中 \(\alpha\) 是一个很小的混合权重。实际中，我们通过在 \(q^\star_e\) 附近扰动机器人初始化，然后要么从扰动后的状态回放原始示范，要么采集一个新的恢复 rollout，来采样第二项。这就在每个接触事件周围形成了受控的局部覆盖，而不是只观察一条穿过该事件的单一标称轨迹。

这种恢复混合占据了图 8 中中央的恢复数据源，并补充了第 4.4 节的聚类平衡采样协议。聚类会挖掘语料库中已经存在的非标称轨迹，而恢复初始化则会主动在那些仅靠聚类不足以提供充分覆盖的接触空间区域中创造这类轨迹。

简要解释：这一节的核心是，除了从已有数据里挑出“异常/非标称”轨迹，还要主动在接触事件附近做局部随机初始化，补齐接触动作空间里的稀疏区域，让模型学到更稳健的恢复与接管能力。

**5.2 VLM 文本条件器预训练**

在这一阶段，我们只适配其 project-out 头和两个轻量级辅助头，而不修改下游 DiT。对当前事件进行一次 VLM 前向传播，并监督它输出两个信号：(i) next-event description，即对即将到来的子任务的 task-decomposition caption；以及 (ii) 对当前事件距离结束还剩多少时间的估计。来自同一次前向传播的隐藏状态将作为 DiT 的条件特征，因此其延迟特性与标准的 T5 conditioning 路径一致。关键的是，VLM 的隐藏状态被对齐到下游 DiT 的 T5 特征空间，使升级后的 VLM 对 DiT 而言可以作为 T5 的即插即用替代。这样的对齐保留了 DiT 侧原有的文本条件先验（DiT 继续以它训练时所采用的几何形式接收文本），同时引入了多模态 VLM 更宽的推理带宽，包括基于场景的歧义消解、任务分解，以及通过剩余时间预测获得的时间锚点。在整个阶段中，VLM backbone 保持完全冻结；只有生成 cℓ 的 project-out 头、next-event 头以及 remaining-time 回归器会被训练。具体地，我们最小化一个三项目标：将 VLM conditioning cVLMℓ（即第 3.4 节中 cℓ 的 VLM 生成版本）对齐到原始 T5 编码器在同一指令上产生的对应 cT5ℓ，同时对两个辅助头进行监督，

### 5.3

Staircase Distillation

第 3.4 节中的 staircase latent CoT 模块通过 Mixture-of-Transformers（MoT）架构实现为一个轻量级推理分支，并与冻结的 Qwen3.5-9B 主干相连。该推理分支以预训练的 Qwen3.5 权重初始化，继承原始自回归模型的语义推理能力，同时支持高效的并行 latent rollout。给定视觉-语言输入，staircase 模块生成一个紧凑的连续 latent 推理状态序列 \(\hat{y}_{1:K_c}\)，这些状态充当隐式推理变量，而不是显式文本 token。这些 latent 状态会在具身 rollout 期间直接注入到 WAM cross-attention 层中。

简要解释：这里描述的是把“链式思维”压缩成连续隐状态，并把它直接送入 WAM 的交叉注意力里，以实现更高效的推理。

用于监督时，latent 序列会通过前缀投影器 \(P_{\text{pref}}\) 投影到冻结的轻量级 Qwen3.5-0.8B 语言模型嵌入空间中的 soft prefix 表示 \(z_{1:K_c}\)，该语言模型以自回归方式重建对应的文本 CoT 轨迹 \(r_{1:M_r}\)：
\[
P_{\phi}(r_{1:M_r}\mid z_{1:K_c}).
\tag{22}
\]

简要解释：这一式子表示用一个更小的冻结语言模型去“读出”latent 推理序列，以重建文字版 CoT，从而让 latent 推理保持可解释性和可监督性。

训练时优化 latent-to-text 重建目标 \(L_{\text{CoT}}\)，梯度只作用于 staircase 推理分支和前缀投影器 \(P_{\text{pref}}\)。在这一阶段，其他所有组件都保持冻结。

简要解释：这说明蒸馏阶段只更新推理分支及其映射层，主体模型不动，以稳定地学习 latent 推理表示。

### 5.4

Next-Chunk Adaptation

在 event-centric 预训练（第 5.1 节）之后，这一可选阶段会在第 3.3 节的 observation-centered 布局上，通过 next-chunk prediction 对 event-pretrained backbone 进行微调。在一个全局指令下，历史帧会对每个 \(H_a\)-step chunk 进行定位，其中 \(H_a\) 表示每个 chunk 中固定的动作步数，即在 event-centric 预训练中使用的可变 event horizon 的固定长度对应物，从而替代 event-centric 预训练期间使用的逐事件 caption。

简要解释：这一段是在说，把按事件组织的预训练模型再适配到按固定步长切块的常规预测任务中，便于统一做固定时长的 VLA 评测。

Observation-centered fine-tuning. 全局指令来自第 4.3 节的 Task level，并由与预训练中相同的冻结 T5 编码；窗口几何结构、相对位姿动作目标，以及扩展的 \(E_{\text{abs}}\) / \(E_{\tau}\) 索引遵循第 3.3 节。两个 DiT towers 都会在这些固定形状窗口上更新，并沿用相同的非对称 1-to-\(N_d\) anchor protocol（\(s^\star=45\), \(m(j)=s^\star\)，在优化期间还可选择性地复用训练时的 \(K=6\) action-noise）。

**内核库（Kernel Library）**  
我们开发了一个定制化的内核库，覆盖训练和推理两个阶段。这一设计的动机来自如下观察：现成的 stock kernels 只在单个算子粒度上工作，因此对于那些在数据流图（dataflow graph）的一个很小邻域内生成并被立即消费的张量（tensor），会不必要地把它们溢写（spill）到全局内存（global memory）。我们识别出一组反复出现、可融合（fusable）的模式；在这些模式的默认实现中，其算术强度（arithmetic intensity）受内存带宽（memory bandwidth）而不是 tensor-core 吞吐量（throughput）限制。我们将这些模式整合为复合内核（composite kernels），使中间激活（intermediate activations）在融合区域内始终保留在寄存器（registers）或共享内存（shared memory）中，从而把有效的 roofline 工作点推向计算受限（compute-bound）区间，并在模型的主要成本中心（dominant cost centers）上持续保持较高的 GPU 利用率。  
简要解释：这一段说明作者通过把多个小算子融合成更大的内核，减少全局内存读写，从而提升 GPU 计算效率和整体吞吐。

我们的内核库构建在 TVM FFI 之上。TVM FFI 是一个与语言无关的 foreign function interface，它通过稳定的 C ABI 暴露已编译内核，并使用 zero-copy tensor handles，从而绕过 PyTorch dispatcher 和 GIL 的逐次调用开销，同时仍然能够与 PyTorch 的训练和推理前端互操作。在这一调度层（dispatch layer）之上，我们提供了一个完整的 wrapper，它与原生 PyTorch op interface 保持一致，因此来自我们库中的内核可以直接替换 PyTorch 对应实现，而不需要对周围的模型代码做任何改动。这样的设计既保留了 host-side launch 的节省，又使得维护训练和推理之间的数值一致性变得简单，因为两条路径都会通过相同的调用面（call surface）触发同一个编译后的内核。  
简要解释：这一段强调库的接入方式很轻量，既能减少 Python/PyTorch 调度开销，又能让训练和推理共享同一套内核实现，保证一致性。

**细粒度重叠（Fine-Grained Overlapping）**  
我们在原始的 spatial-temporal attention 之外引入了 view attention，从而使每一层包含两个顺序执行的 attention。在 Ulysses 序列并行（Ulysses sequence parallelism）下，每个 attention 都需要一对 all-to-alls，来在 sequence 维度和 head 维度之间重新分配序列，因此朴素实现会在关键路径（critical path）上串行执行每层四次 all-to-alls，并显著抬高加入 view attention 的通信成本。为缓解这一问题，我们采用细粒度调度（fine-grained scheduling），将前后紧邻的通信隐藏在 attention 计算之中，从而消除了大部分新增的通信开销。

**多事件序列打包（Multi-Event Sequence Packing）**  
一种常见的训练流程是先按整段 episode 粒度离线编码并缓存视频 latent，然后在优化过程中从这些缓存中采样。每当形成一个 minibatch 时，流水线就必须加载并重新物化（re-materialize）整段 episode 的 latent（或者在线重复执行 VAE 编码）；由于 episode 长度差异很大，若不进行激进的 padding 或 truncation，episode 就无法被打包成统一张量，而有效 batch size 在实践中常常会塌缩。为此，我们改为在 dataloader 层把多个 event 打包成一条更长的序列，并在 packed 序列上并行训练，同时使用 attention mask 阻止跨 event 泄漏（cross-event leakage）。事件会被连接到一个固定的总长度，因此每个训练步都会以完整的配置有效 batch size 运行，GPU 也会尽量保持在计算受限区间；每步成本被摊销到所有打包的 event 上，而不是按每个 episode 单独支付一次。

**模型压缩（Model Compression）**  
世界模型（world models）的部署成本众所周知地高：其底层 diffusion-style backbone 在每一帧生成时都需要执行数十步去噪（denoising）步骤，而每一步本身又涉及一个拥有数十亿参数（multibillion-parameter）的网络的完整前向传播（forward pass），因此朴素部署在根本上无法满足实时机器人控制（real-time robotic control）的延迟预算（latency budgets）。为了弥合这一差距，我们沿着两个正交方向（orthogonal axes）应用两种互补的压缩技术——用 distillation 减少去噪步数，以及用 FP8 quantization 降低每一步的计算和内存成本。

**FP8 量化（FP8 Quantization）**。第二个压缩轴针对的是每一步的计算与内存成本，而不是步数本身；我们将蒸馏后 student 中占主导的矩阵乘法采用数值量化，转换为 8-bit floating-point 精度。FP8 非常适合这一场景，其 exponent-mantissa（指数-尾数）拆分能够保留重尾分布激活值所需的动态范围；对于 INT8 之类的整数格式，即便采用复杂的校准或离群值处理，通常仍会带来明显的精度损失。现代加速器在硬件层面进一步放大了这一优势，通过在更低精度下复用同一条 Tensor Core 数据通路，FP8 矩阵乘法在当前一代 GPU 上的吞吐量大约比 BF16 高 2 倍，同时还将权重加载带宽压力减半，而权重加载成本在机器人控制回路这类典型小 batch size 场景下往往主导推理时延。具体而言，我们对 FP8 采用带 per-block scaling（按块缩放）的 post-training quantization（PTQ，训练后量化）：权重和激活值沿 reduction dimension（归约维度）划分为固定大小的块，每个块分配一个独立的缩放因子，用以吸收局部幅值变化，同时使每个 tensor 的元数据开销保持可忽略不计。这种粒度在量化保真度与运行时效率之间取得了较好的折中：更粗的 tensor-wise scaling（按张量缩放）无法跟踪 transformer 激活值特有的 channel-wise outliers（通道级离群值），而更细的 per-element（逐元素）方案会引入反量化开销，从而侵蚀 FP8 原本希望带来的吞吐优势。

简要解释：这一段说明为什么选择 FP8，以及为什么要用按块缩放的 PTQ。核心是把计算和显存带宽都压下去，同时尽量不损失精度，并且利用现代 GPU 对 FP8 的硬件支持获得实际吞吐收益。

一个天真的 per-block FP8 量化实现会抹去这些吞吐提升中的大部分，因为单独计算 per-block scales（按块缩放因子）以及在运行时对 tensor 做类型转换的成本，相对于 GEMM 本身并不低。为此，我们从两个方面谨慎实现量化算子：权重在离线阶段就被量化为预打包（pre-packed）的 FP8 tensor，并把其缩放因子一并固化进去，因此运行时路径不再承担权重侧量化成本；而激活值的即时（on-the-fly）量化则融合到前一算子的 epilogue（尾部收尾阶段）中，在产生激活值的同一个 kernel 里完成 per-block scale 计算并转换为 FP8，从而避免对 tensor 进行单独的读/写遍历，并把量化开销降低到 GEMM 总时间中可忽略的比例。

简要解释：这里强调“实现方式”决定实际收益。若量化时每步都额外做很多缩放、搬运和转换，理论上的 FP8 加速会被抵消；所以作者把权重量化前置离线完成，把激活量化融合进同一个 kernel 里，最大限度减少额外开销。

简要解释：这一段给出整体部署结果。除了量化本身，还要减少 CPU/host 发起 kernel 的额外开销；综合优化后，系统达到了闭环控制可用的 10Hz 推理频率。

泛化

泛化（Generalization）的设计目标，是在复杂桌面场景中评估 instruction-conditioned transfer，而不是在彼此隔离的、任务专用的重置环境中评估。评估时，场景中会保留多个物体，并以随机顺序发出不同指令，因此策略需要在共享场景状态（shared scene state）下，将当前命令落地到具体对象。任务包括 Place Plates into Storage Slots、Cover Pot with Lid、Push Cleaning Cloth to Table Edge 和 Insert Screwdriver into Cup。该设置旨在考察场景级落地（scene-level grounding）、指令切换（instruction switching）、对干扰物的鲁棒性（robustness to distractors）以及组合泛化（compositional transfer）。机器人必须避免对固定任务顺序过拟合，而应在视觉遮挡和共享物体上下文中，根据当前指令选择相应动作。

**8 讨论**

关于规模、时延与泛化目标。我们当前的 WALL-WM 系列模型规模覆盖了从 10B 参数以下到数百亿参数的范围。跨越这一范围，我们观察到一个稳定趋势：更大的模型能同时提升动作精度和分布外泛化能力，尤其是在需要精细接触时序、长时程状态跟踪以及组合式指令对齐的任务上。这一趋势引出了 embodied foundation models 的一个更广泛问题。该领域过去常常把实时延迟（real-time latency）视为第一约束，因为机器人部署最终需要闭环执行。然而，如果过早强加时延约束，模型可能会在学到足够通用的 world-action prior 之前，就被限制住可达到的性能上限。因此，我们认为，对通用 WAMs 而言，规模驱动的泛化是主要前沿，而时延降低则是一个相对更明确的工程问题。一旦有了强模型，就可以通过蒸馏（distillation）、量化（quantization）、投机式或流式执行（speculative or streaming execution）以及系统级重叠（systems-level overlap）逐步降低时延；但若模型规模不足，想要再把丢失的泛化上限找回来，就要困难得多。

简要解释：这一段强调，先把模型做大、把泛化能力做好，比过早追求低时延更重要；时延可以后续通过工程优化解决，但泛化能力一旦不足很难补回来。

不把成功率作为唯一评估目标。真实机器人的成功率仍然是衡量 embodied competence 最直接的指标，但它也是一个粗糙且噪声较大的度量。一次失败可能由感知、语言对齐、接触动力学、硬件差异、重置质量或环境随机条件等因素引起，而且这些原因在不同任务或不同评测轮次中的分布并不均匀。因此，仅靠成功率对于大规模预训练的分辨率有限：它代价高、速度慢，而且往往难以识别模型究竟哪一部分得到了改进。为此，我们正在探索更适合预训练规模迭代的高效评测协议，包括更细粒度的逐事件（per-event）评分、更密集的进度度量、评测过程中的人工纠正与重标注，以及能够基于生成视频、动作轨迹和中间状态来预测机器人表现的通用评测模型。这样的评测器并不需要完全取代真实硬件。它只需要与下游机器人性能保持可靠的正相关关系，这样就可以在把最有希望的候选方案交到真实机器人上验证之前，先低成本筛选模型与数据决策。

简要解释：这一段指出，机器人成功率太粗糙，不适合作为唯一指标；更好的做法是使用更细的评测方式，先快速筛掉差方案，再用真实机器人做最终验证。

以事件为中心的指令泛化，作为通往 agents 的接口。我们将逐事件的文本泛化（per-event textual generalization）视为连接 embodied control 与更广义 agentic intelligence 的桥梁。当前的 embodied foundation models 往往把可训练参数花在理解和推理高层指令上，但前沿多模态模型和 agents 的快速进展 (57, 17, 1) 迫使我们思考一个边界问题：一个高频操作模型，是否应该把参数竞争在更高层级、而通用 agent 已经能够完成的任务推理上？在许多 embodied benchmark 的设计中，这个边界仍然没有被明确说明。我们的偏好是：先把高层任务的信息分解为逐事件的文本描述，再从逐事件语言进一步映射到视觉与动作。按照这种分解方式，围绕语义事件的全域动作-文本指令泛化（full-domain action-text instruction generalization）应成为 embodied model 的最高优先级目标，而过度关注上游 agent 的推理成本或时延，反而可能成为 embodied intelligence 出现的瓶颈。

简要解释：这一段主张把高层任务拆成事件级文本来处理，让专门的机器人模型专注于动作与视觉对齐，而把复杂推理更多交给上游通用 agent，从系统结构上更高效。

简要解释：这一段是在解释为什么选择 native-T2V 而不是 native-I2V。核心理由是：T2V 提供语义锚点和更强的高层先验，而 I2V 更像像素级外推，较难学到意图、物体身份和子任务边界等高层结构。

(iii) 潜在动作（Latent-action）与双塔视频 + 动作（dual-tower video + action）。这两种范式乍看之下似乎相互对立：一种把动力学压缩进一个显式的、通常是离散的动作代码中，而另一种则把两个 DiT 规模的塔并排放置。然而，仔细看会发现它们在趋同。潜在动作方法会显式地给下一观测值加上一个瓶颈，把它压缩成一种与视觉对齐的、隐式的动态动作代码，而下游策略必须对其进行解码；而双塔 video + action 设计，一旦跨塔共享容量（层级耦合 CA、AdaLN 调制、共享的帧索引 PE）与塔私有容量（每塔的 FFN 宽度、仅动作的状态 cross-attention）之间的比例落在正确的工作点上，其执行的压缩本质上是相同的。共享子空间就是一种 latent action，只不过它是涌现出来并通过端到端学习得到的，而不是通过一个必须提前猜测其宽度和 codebook 的架构瓶颈强加出来的。因此，双塔表述更适合被理解为 latent action 的一种更宽松形式，而不是它的竞争对手：收紧共享子块会恢复任何显式瓶颈范式，而放宽它则在真实动力学超出任何手工选定的潜在维度时，为额外宽度留下空间。形式化地，设 \(h^{shared}_t \in \mathbb{R}^d\) 表示 V 信息传入 A 塔所经过的跨塔瓶颈激活，设 \(z_t\) 表示 LAPA 风格 encoder/codebook/decoder 的显式 latent-action code；当 \(d\) 与 codebook 宽度一致时，这二者承载的是同一个信息瓶颈，除了 \(z_t\) 的离散性之外可以互换（Figure 20b）。这三项选择共同把 WALL-WM 放在了像素空间初始化、原生 T2V 目标，以及一个刻意设定共享/私有容量大小的双塔结构上。

9.2  Wan VAE 中的时间鲁棒重建

一个实际问题是，Wan VAE 的时间下采样是否依赖强的短程时间相关性。我们用刻意对抗性的输入来压力测试这一假设：在时间上相隔很远采样的帧，甚至把异质的静态图像沿时间轴堆叠起来，假装它们组成了一段视频。VAE 仍然能够成功重建这些序列，这表明它的压缩并没有严重偏向于利用局部时间冗余。这个特性对 WALL-WM 有利：即使是跳帧的具身视频，也不必看起来像平滑的 web-video 片段，VAE 仍能保留逐帧的视觉内容。

简要解释：这一段想证明 Wan VAE 的压缩不只是“吃”相邻帧的重复信息，即便输入的视频时间上不连续、甚至像拼接出来的伪视频，它也能较好恢复，因此更适合处理跳帧的具身数据。

9.3  继承而来的 Wan 特征的视觉-语义与空间能力

本小节解释为什么继承而来的 Wan 特征位于条件栈（conditioning stack）的底部，作为一个不可移动的底座，而不是又一个可以替换的 encoder。尽管 Wan 是用视频生成目标训练的（而不是判别式目标），但它的 hidden-state 几何结构同时编码了 (i) 物体/实例级的视觉-语义结构，以及 (ii) 3D、与相机一致的空间结构。在空间这一轴上，它实际上比经典的对比式图像预训练特征更强。

跨视角 PCA：一个直接的视觉诊断。我们在部署相机架构中的同步 ego / left-wrist / right-wrist 三元组上验证这一点。我们从三个视角中提取同一个去噪步上的中间 Wan DiT 特征，用在整个三元组上联合拟合的共享 PCA 基底进行投影（Figure 21，中间行），再把得到的 PCA 着色叠加回原始帧上（底行）。结果出现了三种模式。第一，语义等价的表面——被操作的物体、操作者的手，以及空碗——会塌缩到少量 PCA 分量上，并且在不同视角之间呈现一致的颜色，尽管这三个摄像头的原始像素重叠很少。第二，这种着色跟踪的是底层物理表面，而不是依赖于特定视角的图像梯度：同一个物体在 left-wrist 与 right-wrist 的相机位姿变化下，基本保留其 PCA 特征。第三，对于任意一对视角，逐 token 的跨视角 cosine 一致性都保持在约 \(\sim 0.97\)（见 Figure 21 中的 meta-data strip）。在相同协议下，从同一三元组中提取的 ViT-SigLIP 和自监督图像特征会稳定地产生更低的一致性；这种差距主要集中在对几何敏感的 token（物体边缘、接触区域）上，而不是背景上。

简要解释：这一段说明 Wan 的特征不仅懂“是什么”，还懂“在哪里、怎么摆放”，而且这种跨视角几何一致性很强。作者用跨视角 PCA 和 cosine 一致性证明，Wan 特征比一些常见图像特征更适合做具身任务的条件底座。
