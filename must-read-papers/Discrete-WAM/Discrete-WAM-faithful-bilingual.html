摘要

自动驾驶要求对自车动作如何塑造周围世界的演化进行推理。然而，大多数端到端方法依赖直接的状态到动作映射，只捕捉相关性，而没有显式建模动作条件下的动力学。相反，连续潜变量世界模型（continuous-latent world models）往往缺乏用于跨反事实未来进行因果推理的组合式结构。我们提出 Discrete-WAM，一种统一的潜在视觉-动作世界策略（latent vision-action world policy），将未来视觉状态和自车动作表示为对齐的离散 token，从而能够在不同未来之间进行组合式因果推理。基于这种统一的离散对齐，Discrete-WAM 构建了一个共享的离散扩散框架（shared discrete diffusion framework）与统一的生成任务（unified generative tasks），联合形式化 world modeling、world-action policy 以及 hierarchical decision-enabled policy，支持跨多种驾驶场景的组合式泛化。大规模自动驾驶基准实验表明，Discrete-WAM 在支持可控生成与反事实推理的同时，能够取得有竞争力的性能，为更可靠的决策制定提供了一条有原则的方法路径。

1 引言

自动驾驶从根本上要求对自车动作如何塑造未来世界演化进行推理，而不仅仅是对瞬时观测做出反应 [12, 38, 40]。现有的端到端（E2E）自动驾驶系统 [13, 28, 47, 49, 53] 通常通过行为克隆 [11] 将驾驶形式化为从视觉到动作的直接映射，只是捕捉统计相关性，而没有显式建模动作条件动态。尽管引入预测能够实现显式的未来推理 [25]，但对预定义的中间表示和标注 [15, 31, 80] 的依赖限制了泛化能力。近期的视觉-语言-动作（VLA）[43, 45, 85, 103] 模型通过语言监督融入了更广泛的语义先验，但其推理能力仍然高度依赖标注的语义抽象，这充当了建模细粒度时空动态的低带宽接口 [34]。这些局限表明，自动驾驶系统在表示形式与生成建模上存在更根本的瓶颈。  
简要解释：这一段指出，当前自动驾驶系统的核心问题不只是“看见后反应”，而是缺少能把动作、未来和语义统一起来的表示与生成机制。

一般而言，端到端自动驾驶系统运行在连续潜空间中 [26, 51, 97]，其中表示高度纠缠，虽然具有很强的连续性和插值能力，但缺乏显式的组合式语义 [56]。因此，观测、动作和未来状态在潜空间中仍然只是弱对齐的，这限制了可靠的动作条件推理，以及针对不同可能未来的反事实比较。相比之下，离散表示天然提供可组合的语义单元，可作为视觉、动作和未来演化之间共享的锚点 [42, 60, 94]。通过学习统一的离散词汇表，观测、自车动作和未来状态可以在同一个对齐的语义空间中表示，从而实现显式的视觉-动作对齐，以及对动作条件未来演化的结构化建模。  
简要解释：这一段强调，离散表示比连续潜变量更适合把“看见什么、做了什么、会发生什么”对应起来。

因此，核心挑战不仅仅是生成未来，而是学习观测、动作及其所诱发未来演化之间的结构化对应关系 [32, 78]。除了表示之外，自动驾驶从根本上还需要对动作条件未来进行生成建模 [88]。连续扩散模型提供了全局一致的生成能力，但它们仍然工作在无结构的连续潜空间上，缺乏显式的语义落地 [2, 49, 98]。自回归（AR）形式引入了离散的组合式生成，但其推理过程仍然是以前缀为条件的，受限于一种归纳偏置，因而限制了全局一致性或目标驱动的推理 [3, 10, 103]。  
简要解释：这一段说明，单纯会“生成”还不够，关键是要生成得有结构，并且能和动作选择及目标一致。

与此同时，现有世界模型、世界-动作建模以及视频预测方法通常被优化为附属目标，或作为外挂模块接到规划器上 [43, 44, 75]。尽管这些方法在预测与策略学习之间共享表示，但未来建模和决策制定仍被当作分离的任务，而不是一个统一的生成过程 [29, 30, 46]。因此，现有方法难以在观测、动作和由此诱发的未来状态之间建立一致的因果对应关系，从而限制了对长时程交互式未来的全局一致推理。  
简要解释：这一段指出，预测和决策如果分开做，就很难形成一致的因果理解，更难处理长时序交互。

在本文中，我们认为，要从反应式模仿走向结构化的世界-策略推理，三个关键属性是必要的：(i) 对齐的离散表示，(ii) 用于未来观测和动作的统一生成建模，以及 (iii) 将高层决策与低层动作实现分离的层级式策略建模。基于这一观点，我们提出 Discrete-WAM，这是一个统一框架，将自动驾驶表述为在共享离散 token 空间上的序列建模，其中视觉观测、自车动作、未来状态和决策变量都被表示为离散 token，并通过离散扩散架构联合建模。Discrete-WAM 不把动作仅仅当作未来预测的条件信号，而是将观测、动作、决策和未来演化建模为统一生成过程中的耦合变量，从而实现世界状态与驾驶决策之间的双向推理。  
简要解释：这一段给出了本文的方法思想：把感知、动作、决策和未来统一到同一个离散生成框架里。

Discrete-WAM 的一个核心组件是结构化策略建模。不同于直接从场景上下文生成稠密的未来动作，Discrete-WAM 将策略生成分解为一个层级式决策-规划过程：模型首先预测一个稀疏的高层决策骨架，用来捕捉粗粒度的机动意图和多模态驾驶选择，然后在该决策条件下执行基于离散扩散的动作 token 编辑。这一设计把多模态决策选择交给高层骨架，把平滑轨迹实现交给低层动作规划器，使策略解码器能够生成时间上一致的动作，而不会塌缩到单一的驾驶模式。  
简要解释：这一段说明策略不是一步直接吐出，而是先定“做什么”，再细化“怎么做”，从而保留多种驾驶可能性。

2 架构

2.1 Discrete-WAM 的基础架构

Discrete-WAM 架构由四个主要组件组成：(1) 一个视觉 VQ Tokenizer，它将视觉观测编码为离散语义 token，并通过一个 projector 将视觉特征对齐到 Transformer 的隐藏维度；(2) 一个上下文编码器，它将自车状态和导航指令注入到潜在表示中。  
简要解释：这一段在概述模型的输入组织方式：先把图像离散化，再把车辆状态和导航信息作为条件加入，让后续 Transformer 能统一处理多种信息。

**2.2 视觉标记化（Vision Tokenization）**

视觉标记器将连续的相机观测转换为紧凑的离散视觉 token，在原始图像与 Transformer 主干之间提供 token 级接口。该离散表示使视觉观测能够以与动作 token 相同的序列格式被处理，同时保留下游世界建模和策略建模所需的场景语义。为了获得紧凑的视觉表示，我们遵循先前工作 [93]，并预训练一个基于 VQ-VAE 的标记器 [72]，将前视相机图像编码为离散视觉 token。  
简要解释：这一节说明如何把图像压缩成离散 token，既保留语义信息，又能与动作 token 统一建模。

**2.3 动作标记化（Action Tokenization）**

对于动作表示，我们将连续的未来运动转换为与离散 token 兼容的表示，并基于加速度词表（acceleration vocabulary）进行建模。给定一个跨越 H 个时间步的未来轨迹，我们首先用三次样条（cubic spline）拟合离散轨迹，以得到平滑的连续曲线。随后，我们使用二阶有限差分计算每个未来步的以自车为中心（ego-centric）的二维加速度，记为 (ax , ay )，其中 ax 和 ay 分别对应纵向加速度和横向加速度。样条拟合步骤改善了所导出加速度序列的连续性，并确保对恢复出的加速度进行积分后能够与原始离散轨迹高度一致。  
简要解释：这一段是在说，先把未来轨迹平滑化，再转成纵向/横向加速度，方便后续离散化并尽量不损失轨迹信息。

我们通过分别将 ax 和 ay 的有效范围划分为 Nx 和 Ny 个区间，构造一个均匀分布的二维加速度词表。这些区间的笛卡尔积形成一个网格结构的动作词表，其大小为 Nx × Ny。我们不把每个连续加速度向量简单地分配到最近的单个区间，这样会引入确定性的硬量化误差；相反，我们使用其邻近词表项上的软目标（soft target）来表示它。具体来说，对于每个加速度分量，我们找到夹住该连续值的两个相邻区间中心，并根据该值在两者之间的相对位置分配插值权重。在二维加速度词表中，这会在 (ax , ay ) 周围的四个邻近原型上产生一个软标签（soft label）。在这种网格插值下，连续加速度在每个加速度网格单元内都具有精确的插值表示。  
简要解释：这里把连续加速度变成网格上的“软分布”而不是硬分类，从而减少硬量化带来的误差。

在训练期间，动作头（action head）使用交叉熵（cross-entropy）对这种软目标分布进行优化，而不是对 one-hot 标签进行优化。在推理时，预测得到的动作分布可以通过对加速度词表做加权求和映射回连续加速度。在软目标分布被精确恢复的情况下，连续加速度可以被重建为相邻词表原型的加权和。因此，所提出的软标签表示消除了确定性的硬分配量化误差，而剩余的重建误差则归因于分布预测不匹配。相应的推导和误差界在附录中给出。  
简要解释：训练时学的是软分布，推理时再加权还原成连续值；这样主要减少的是离散化本身造成的误差。

因此，一个未来动作序列被表示为离散加速度词表上的 At+1:t+H，而连续控制值则通过软标签插值得以保留。这种动作标记化使视觉 token 与动作 token 能够在同一个 Transformer token 空间中被联合建模，同时避免了连续动作硬量化所引入的确定性误差。  
简要解释：最终目标是把未来动作也变成 token 序列，与视觉 token 一起输入同一模型，同时尽量不损失连续控制精度。

**2.4 统一世界策略（Unified World Policy）**

Discrete-WAM 将自动驾驶表述为一个在离散视觉 token、决策 token 和动作 token 上进行的统一世界-策略建模问题。在时间步 t，我们将场景上下文记为 Ct，它包含历史视觉观测、自车状态信息以及导航指令。未来视觉观测表示为离散视觉 token 序列 Vt+1:t+H，未来动作策略表示为离散动作 token 序列 At+1:t+H，其中每个动作 token 都对应于第 2.3 节中定义的一个量化加速度原型。我们进一步将高层决策条件记为 Dt，它捕捉稀疏的低频驾驶结构，例如机动意图、目标车道、粗粒度航点、速度趋势或交互优先级。  
简要解释：这一节把自动驾驶统一成“世界状态 + 高层决策 + 未来动作/视觉预测”的联合建模问题，便于一个模型同时学规划和预测。

在此记号下，Discrete-WAM 通过一个共享的 token 编辑（token-editing）接口整合了三种训练模式。  
这些模式的区别在于：哪些 token 流被用作条件输入，以及哪些 token 流被视为预测目标。  
第一种模式是世界建模（world modeling），其目标是视觉预测（vision prediction）。给定场景上下文 \(C_t\) 和未来动作 token \(A_{t+1:t+H}\)，模型预测未来视觉 token \(V_{t+1:t+H}\)。该任务训练模型理解在指定动作序列作用下场景如何演化，并为下游策略学习提供动作条件的世界动力学（action-conditioned world dynamics）。  
简要解释：这一模式让模型学会“给定动作，预测世界会怎样变化”，本质上是在学习环境动态。

第二种模式是世界-策略建模（world-policy modeling），它同时训练视觉预测和动作预测。在这一模式中，模型在同一个 token 序列内同时推理未来动作及其引发的视觉后果。动作 token 表示未来策略（future policy），而视觉 token 表示相应的未来世界状态。这样的形式化鼓励模型将策略生成与世界演化耦合起来，而不是把它们作为彼此独立的目标来学习。  
简要解释：这一模式把“要怎么做”和“做了之后世界会变成什么样”放在一起学，强调策略与环境变化的联合建模。

第三种模式是策略建模（policy modeling），其目标是先进行决策预测，再进行动作预测。模型首先从场景上下文 \(C_t\) 中预测高层决策条件 \(D_t\)。在该决策条件下，动作 token 规划器随后预测未来动作序列 \(A_{t+1:t+H}\)。这种层次化分解将策略学习拆成两个层级：高层决策任务捕捉多模态驾驶选择，而低层动作预测任务则专注于在所选决策条件下生成平滑且时间上连贯的轨迹。  
简要解释：这一模式先决定“走哪种路线/做哪类动作”，再生成具体轨迹，适合处理驾驶中存在的多种可行选择。

总之，这三种模式共同提供了一个统一架构，用于动作条件世界预测、联合世界-策略学习，以及决策条件动作生成。其数学任务形式化在第 3 节中描述。

世界建模任务学习动作条件下的未来视觉预测。给定场景上下文 \(C_t\) 和未来动作序列 \(A_{t+1:t+H}\)，模型预测未来视觉 token 序列 \(V_{t+1:t+H}\)。这可以写为

\tag{1}
\]

\[
p_{\psi,\theta}(A_{t+1:t+H}, D_t \mid C_t)
=
p_\psi(D_t \mid C_t)\, p_\theta(A_{t+1:t+H} \mid C_t, D_t).
\tag{2}
\]

这种层次化分解把策略学习分成两个层级：高层决策任务捕获多模态驾驶选择，而低层动作预测任务在所选决策条件下生成平滑且时间一致的轨迹。这里，\(D_t\) 被视为一个稀疏的潜在骨架，它编码低频规划结构，例如粗粒度的机动意图、参考运动趋势或其他决策层约束。  
简要解释：策略建模不是直接一次性生成整段动作，而是先生成“要做什么”的粗决策，再生成“怎么做”的细动作，这样更稳定，也更符合驾驶规划逻辑。

**理论分析。**

引入 \(D_t\) 的动机在于：当只有场景上下文被给定时，未来动作 token 并不是条件独立的。令 \(U \subseteq \{1,\ldots,H\}\) 表示未来步中的一个子集，令 \(A_U=\{A_{t+h}: h\in U\}\) 表示对应的动作 token 组。我们用 \(TC(\cdot \mid \cdot)\) 表示条件总相关性（conditional total correlation），它度量在给定条件下，一组 token 之间残余的统计依赖性。按照附录 7.3.2 的推导，在上游决策骨架条件化之后，残余依赖性满足  
\[
\mathbb{E}_{D_t}\, TC(A_U \mid C_t, D_t)
=
TC(A_U \mid C_t) - R_D(U \mid C_t),
\tag{3}
\]
其中 \(R_D(U \mid C_t)\) 是由决策骨架带来的冗余增益（redundancy gain）。这个等式表明，只有当 \(R_D(U \mid C_t)>0\) 时，决策条件化才会减少残余动作 token 依赖。

简要解释：这里的核心意思是，加入高层决策骨架 \(D_t\) 后，动作 token 之间原本“纠缠在一起”的统计依赖可以被部分解释掉，从而让低层动作建模更容易。

附录 7.3.3 进一步给出了冗余增益为正的充分条件。在残差混合（residual mixing）假设下，经过决策骨架条件化后，未来动作 token 之间的依赖会随着时间距离衰减：  
\[
I(A_{t+i}; A_{t+j} \mid C_t, D_t)
\le
\beta \exp\!\left(-\frac{d(i,j)}{\ell_D}\right),
\tag{4}
\]
其中 \(\beta\) 衡量剩余的局部耦合强度，\(\ell_D\) 表示在决策骨架条件化之后的残余相关长度。这个假设意味着，一旦低频的驾驶意图已被 \(D_t\) 解释，剩下的细粒度动作 token 主要编码局部残差修正。如果原始动作 token 组满足依赖下界 \(TC(A_U \mid C_t)\ge \kappa(U)\)，那么  
\[
\kappa(U) >
\mathbb{E}_{D_t}\!\left[
\sum_{\{i,j\}\subset U}
\beta \exp\!\left(-\frac{d(i,j)}{\ell_D}\right)
\right]
\Longrightarrow
R_D(U \mid C_t) > 0.
\tag{5}
\]

因此，一个有效的决策骨架会在它解释的组级低频依赖强于剩余的、经骨架条件化后的局部依赖时，减少残余总相关性。  
简要解释：这段在说明什么时候“先预测决策骨架”是有价值的。若骨架能解释掉足够多的长期、低频结构，那么剩下的动作细节就会更独立、更容易建模。

当同时考虑决策预测误差和 token 级模型误差时，同样的分析还能得到一个 schedule-level 的 KL 上界。令 \(\pi\) 表示 token-editing schedule，\(A_r\) 表示第 \(r\) 轮的 active edit set，\(S_r\) 表示第 \(r\) 轮编辑前的状态。附录 7.3.4 表明  
\[
D_{KL}\!\left(q(A \mid C_t)\,\Vert\,p_{\psi,\theta,\pi}(A \mid C_t)\right)
\le
\delta_D + \delta_{init} + B_{model}(\pi) + U_{dep}(\pi),
\tag{6}
\]

简要解释：这说明整体生成误差可以分解为决策误差、初始化误差、模型误差和依赖相关的误差几部分，因此 token 编辑日程 \(\pi\) 会直接影响最终 KL 误差上界。

对于世界建模（world modeling），未来动作 token 被视为干净的条件输入，而不是编辑目标。因此，在这一任务中，动作流不使用双重 clean-noisy 填充。未来视觉流以 token-editing 形式构建，模型在历史观测、自车状态（ego-state）和导航 token，以及所提供的未来动作序列的条件下，预测被污染的未来视觉 token。  
简要解释：这一段说明了世界建模时，动作主要作为条件信息输入，模型重点学习根据过去信息和未来动作去恢复/预测未来视觉 token。

对于策略建模（policy modeling），上下文 token 遵循因果注意力（causal attention）结构，因此模型可以在不访问未来上下文的情况下，基于历史观测和可用状态信息进行条件建模。未来动作 token 构成编辑目标块（editing target block），并被允许在动作块内部进行双向注意力（bidirectionally）。这种双向动作注意力使模型能够联合精炼一整段未来动作序列，而因果上下文掩码（causal context mask）则防止来自观测侧的未来信息泄漏。  
简要解释：这一段强调策略建模时，观测侧必须保持因果性，而动作侧允许双向交互，以便整体优化未来动作序列。

对于世界-策略建模（world-policy modeling），未来视觉 token 和未来动作 token 都可以通过 token-editing 监督进行训练。我们为每个可编辑流采用双路径填充顺序（dual-path filling order），其中干净 token 和噪声 token 作为独立块（separate blocks）放置。注意力掩码在干净目标块（clean target block）与对应的噪声预测块（noisy prediction block）之间强制双向隔离：噪声 token 可以使用被允许的上下文和任务特定的可见条件，但不能直接关注其干净目标。在训练期间，teacher forcing 提供历史真实值信息作为上下文，而模型学习将有噪的视觉和动作 token 编辑为其干净目标。  
简要解释：这一段说明世界建模与策略建模统一起来后，视觉与动作都可以用“先加噪再编辑回干净目标”的方式训练，并通过掩码控制信息流。

统一的预训练目标（pretraining objective）将 token 级分类损失与连续运动重建损失结合起来。对于视觉 token 编辑，模型通过离散视觉词表（discrete visual vocabulary）上的交叉熵损失（cross-entropy loss）进行监督。不同于只监督被污染位置的目标，我们将 token 分类损失应用于所有可编辑的视觉 token 位置，包括干净 token 和被污染 token。对于被污染位置，该损失训练模型从噪声输入中恢复原始干净目标。对于干净位置，同样的损失鼓励恒等映射（identity mapping），要求模型保留那些已经接近真实值的 token，而不是不必要地编辑它们。这种全位置监督（all-position supervision）为 token 编辑提供了一个隐式停止信号（implicit stopping signal）：模型不仅学习如何纠正噪声 token，也学习何时不需要编辑。当存在潜在视觉嵌入（latent visual embeddings）时，我们进一步施加潜在重建损失（latent reconstruction loss），以保留超越离散 token 索引的细粒度视觉语义。  
简要解释：这一段说明视觉部分的训练目标不仅要“改对被污染 token”，还要学会“别乱改已经正确的 token”，并可进一步保留更细致的语义信息。

训练目标（Training objectives）。

对于动作 token 编辑，模型通过离散动作词表（discrete action vocabulary）上的交叉熵损失进行监督。除了 token 分类之外，我们还施加运动级监督（motion-level supervision），以确保离散动作分布与连续未来运动在物理上保持一致。首先，我们监督与预测动作分布相关联的解码加速度（decoded acceleration），以鼓励离散动作 token 保留准确的底层运动语义。其次，我们将预测的动作分布转换为连续加速度，并对其在时间上积分两次，以重建未来自车轨迹（ego trajectory），并在该轨迹上应用轨迹级回归损失（trajectory-level regression loss）。此外，我们还引入一个针对积分后未来位置的辅助因子分解位置分类损失（auxiliary factorized position classification loss）。这个辅助任务分别监督纵向和横向坐标，并为由预测动作 token 所诱导的轨迹提供细粒度空间约束。词表配置和损失公式的详细内容见 Appendix 7.2。  
简要解释：这一段说明动作不仅要在离散 token 层面预测正确，还必须在连续轨迹层面保持物理合理性，因此同时使用分类、轨迹回归和位置分类等多种监督。

一种获得用于运动重建的连续加速度的朴素方法，是将预测动作概率与加速度词表相乘，并使用得到的全分布期望（full-distribution expectation）。然而，动作预测通常天然是多模态的（multi-modal）：不同的加速度模式可能对应不同但都合理的驾驶决策。直接对整个预测分布取期望，可能会把彼此不兼容的模式平均掉，从而产生物理上不合理的中间加速度。我们将这一问题称为 decoding-induced mode averaging。  
简要解释：这一段指出，如果直接对多模态动作分布求平均，可能会把多个合理驾驶模式“平均坏了”，导致生成的加速度不真实。

为缓解这一问题，我们仅在构建连续加速度、轨迹重建以及辅助位置监督损失时，使用一种 mode-aware decoding 策略。我们用多模态高斯混合模型（Gaussian mixture model, GMM）拟合加速度词表上的预测类别分布。具体而言，我们考虑包含 1、2 和 3 个分量的 GMM，并选择拟合误差最低的那个。随后，我们使用 top-p sampling 选择一个高斯模式（Gaussian mode），在所选模式内部重新归一化动作概率，并基于重新归一化后的分布和加速度词表计算期望加速度。得到的 mode-aware 加速度随后用于轨迹重建的时间积分。  
简要解释：这一段说明作者用“先识别主导模式、再在该模式内计算加速度”的方式，避免多模态平均带来的不合理运动。

3.2  
后训练（Post Training）

为进一步捕获数据集中由于覆盖有限而难以学到的罕见行为或安全关键行为，我们引入后训练（post-training）微调阶段，利用基于模型的轨迹采样（model-based trajectory sampling）和强化学习（reinforcement learning）在困难场景上优化策略，同时保留先前学到的行为。  
简要解释：后训练的目标是补足长尾场景和高风险场景，让策略在难例上更稳健，而不是只依赖原始数据分布。

对于每个驾驶场景上下文 \(C_t\)，Discrete-WAM 通过 token-edit planner 生成一组候选轨迹 \(\{\tau_1,\ldots,\tau_G\}\)，用于高效探索。每条轨迹都会通过 \(r\) 轮迭代编辑：
\[
\tau_i^{(r)} \sim p_\theta(\hat A_{t+1:t+H}\mid \tilde A_{t+1:t+H}, \hat D_t, C_t)\,p_\psi(\hat D_t \mid C_t).
\]
这会产生一组多样且高质量的 rollouts，并用在线奖励函数 \(R(\tau_i)\) 进行评估，即 \((E)/PDMS metrics\)。对 \(D_t\) 的分层建模还支持两种采样策略。1) 在最可能的决策 \(\arg\max p_\psi(\hat D_t \mid C_t)\) 下进行组采样（Group sampling）。2) Discrete-WAM 还可以直接利用决策 \(\hat D_t\) 的完整分布，并针对每个 decision token 分别执行组采样。这进一步支持对 \(\log p_\psi(\hat D_t \mid C_t)\) 进行 decision-level post training update。  
简要解释：这里通过“先选决策、再采样轨迹”的方式生成多种候选未来，并用奖励函数筛选；同时也保留对整个决策分布的训练更新，从而兼顾探索和分层决策学习。

策略采样（Policy sampling）。

遵循 Grouped Relative Policy Optimization（GRPO）范式，我们使用 following [101] 的 one-step reconstruction estimator，计算当前策略 \(\pi_{\theta,i}^A\) 下逐 token 的对数概率：
\[
\sum_{h=1}^{H}\log p_\theta(\hat A_{t+h}\mid \hat A_{<t+h}, \hat D_t, C_t)
\]
决策分布则直接汇总为 \(\pi_{\theta,i}^D \sim \log p_\psi(\hat D_t \mid C_t)\)。优势值（advantage）计算为
\[
A_i = R(\tau_i) - \frac{1}{G}\sum_{i=1}^{G} R(\tau_i),
\]
整体目标变为：  
\[
\mathbb{E}_{\tau \sim \pi_{\theta,\psi}}
\left[
\frac{1}{G}\sum_{i=1}^{G}
\min\!\Big(
\rho_i^k A_i^k,\,
\text{clip}(\rho_i^k, 1-\epsilon, 1+\epsilon)A_i^k
\Big)
- KL(\pi_{\theta,\psi}\,\|\,\pi_{\text{ref}})
\right],
\tag{10}
\]
\[
k \in (A,D).
\]

## 4 评估

### 4.1 设置

**数据集与基准。** 我们在 NAVSIM-v1 和 v2 基准 [8, 17] 上展示 Discrete-WAM 的端到端（E2E）生成与规划能力，该基准提供了用于端到端驾驶的大规模驾驶场景。按照标准协议，我们在 navtest 划分上评估 Discrete-WAM，该划分包含 1.2 万个以 2 Hz 采样的驾驶场景。NAVSIM 使用 PDMS 和扩展版 EPDMS 评估 4s 视野轨迹的规划质量，其中安全和规则关键指标作为乘性约束纳入，而进度和舒适性相关项则通过加权聚合组合。报告的指标包括无责任碰撞（NC）、可行驶区域符合度（DAC）、行驶方向符合度（DDC）、交通灯符合度（TLC）、自车进度（EP）、碰撞时间（TTC）、车道保持（LK）、历史舒适性（HC）以及扩展舒适性（EC）。详细的指标公式和基线见附录 7.2.4。

5

结论与未来方向

在这项工作中，我们提出了 Discrete-WAM，一个面向自动驾驶的统一离散视觉-动作世界-策略框架（unified discrete vision-action world-policy framework）。Discrete-WAM 将未来视觉状态、驾驶决策和自车动作形式化到一个共享的离散 token 空间中，并通过统一的 token 编辑范式（token-editing paradigm）对其进行联合建模。一个离散扩散框架（discrete diffusion framework）在共同的生成目标下，将世界建模（world modeling）、世界-策略建模（world-policy modeling）以及分层的决策条件策略学习（hierarchical decision-conditioned policy learning）整合到一起。我们在 NAVSIM benchmarks 上的大量实验表明，Discrete-WAM 在实现强规划性能的同时，还支持可控世界生成、反事实推理以及面向安全的策略改进。  
简要解释：这一段总结了本文的核心贡献：把“看见什么、怎么决策、怎么行动”放进同一个离散表示空间里联合学习，从而同时提升规划能力、可控生成能力和安全性。

展望未来，仍有几个很有前景的方向。首先，是构建一个完全统一的世界-动作基础模型（world-action foundation model），能够支持长时程交互式仿真和策略学习。其次，更丰富的层次化抽象（hierarchical abstractions）与语言条件化目标（language-conditioned objectives）可能进一步提升规划的多样性和可控性。第三，基于奖励的后训练（reward-guided post-training）可以推广到轨迹级监督之外，从而在稀疏或安全关键反馈下实现更有效的策略优化。最后，统一的离散 token 空间为通过想象 rollout（imagined rollouts）、反事实推理和测试时规划（test-time planning）实现可扩展自我改进提供了机会，并可能为通用具身决策系统（general-purpose embodied decision-making systems）提供一条路径。  
简要解释：这一段提出了后续研究方向，重点是把当前框架进一步扩展为更通用、更长时程、能结合语言和奖励进行优化的基础模型。

7.1

7.1.1

世界策略建模

世界模型（World models, WMs）已成为物理智能的核心范式之一，其目标是建模在智能体动作作用下外部世界如何演化[23]。在自动驾驶领域，早期受世界模型启发的方法通常会借助中间表示与规划相结合，例如占用预测（occupancy forecasting）[53, 71, 80] 或联合运动预测（joint motion prediction）[15, 25, 31, 52]，其中未来动力学主要被学习出来以服务下游决策。随着大规模视觉基础模型（vision foundation models）[38] 的出现，近期研究越来越转向未来视觉监督（future visual supervision）[21, 57, 77, 89]，利用图像和视频生成作为丰富的监督信号，用于表征学习（representation learning）[75]、面向规划的推理（planning-oriented reasoning）[91]，甚至显式或隐式的奖励建模（reward modeling）[44, 79]。与此同时，另一条研究路线聚焦于仿真和数据生成引擎。这类方法通过结构化世界建模（structured world modeling）[90] 构造交互式或长尾场景，例如行为仿真（behavior simulation）[50]、3D 场景重建（3D scene reconstruction）[20, 70]，或生成式视频模型（generative video models）[77, 88]。通过合成多样化的未来结果和反事实交互（counterfactual interactions），它们为策略评估、闭环训练和安全验证提供了可扩展环境。尽管这些方法已取得成功，但现有大多数方法仍将世界建模与规划视为松耦合组件，即世界模型主要充当辅助预测器或模拟器，而不是在统一因果表述下与决策过程联合优化。

简要解释：这一段在说明世界模型研究的两大方向，一类偏重预测未来世界状态，另一类偏重仿真与数据生成，但二者大多没有真正与策略学习统一起来。

这一局限促使近期出现了世界-动作建模（world-action modeling, WAM）[104]，其将未来观测与策略生成联合表述为一个统一的学习问题[9, 43]。有些工作将未来世界与智能体动作建模为一个单一生成过程[59, 82, 95]，另一些工作则强调统一预训练（unified pretraining）[96]、共享的世界-策略表征（shared world-policy representations）[5]、潜在动作建模（latent action modeling）[22]，以及感知价值的世界模型（value-aware world models）[37]，以减少显式观测生成的冗余，并加强环境动力学与控制之间的耦合。然而，现有表述大多建立在连续潜在空间之上，这通常会带来表征歧义（representation ambiguity）[66, 98]。这些局限最近又推动了基于离散 token 空间的并行生成范式被采用，包括掩码建模（mask modeling）和离散扩散（discrete diffusion）[73, 84, 99]。虽然这类方法提高了生成效率并支持迭代细化，但其中大多数仍将决策建模为从观测到动作的直接映射，而没有显式学习覆盖未来世界与策略的统一生成先验。与此形成对比的是，Discrete-WAM 将观测与动作置于共享的离散表示空间中进行建模。统一生成预训练在世界演化与策略生成之上建立共同先验，而联合离散扩散表述则为世界建模和决策提供了统一框架。

简要解释：这一段强调，从 WAM 发展到 Discrete-WAM 的关键动机是把世界演化和动作生成放到同一个离散生成空间里，从而真正共享先验并统一优化。

**7.1.3**

**策略后训练（Policy Post-training）**

尽管大规模预训练已经显著提升了 E2E 自动驾驶系统的泛化能力 [11]，但其本质上是在 open-loop 目标下优化，因此无法完全应对 closed-loop 交互所引入的分布偏移（distribution shifts）[76]。特别是，behavior cloning 只能学习去匹配示范动作，却不会直接优化 closed-loop 评测中使用的序列目标，从而造成持续存在的目标差距（objective gap）[35]。为缓解这一差异，研究者探索了离线强化学习（offline reinforcement learning）和用于策略对齐（policy alignment）的后训练策略 [48, 68]。早期方法构造正、负轨迹对，并应用偏好优化（preference optimization），例如 DPO 风格排序（DPO-style ranking）[64] 或逆强化学习（inverse reinforcement learning）[33]，以鼓励更优行为。然而，这些方法优化的仍然是相对偏好，而非长时程驾驶目标。最近，基于策略梯度（policy-gradient-based）的方法，例如 GRPO [65]，被引入以直接优化轨迹级奖励（trajectory-level rewards）和 closed-loop 指标 [44, 45, 54]。不过，现有方法通常作用于完整轨迹，因此只是在固定决策条件下改进轨迹生成，而没有显式优化高层决策过程本身。另一条并行研究路线利用基于 3DGS 的模拟器或学习到的世界模型（world models），通过交互来实现在线 RL [20, 50, 70, 86]。尽管这些方法提供了 closed-loop 优化的机制，但其性能本质上会受到累积的仿真误差和模型误差限制，这会给策略学习引入额外偏差。最近的方法尝试通过直接在潜在转移空间（latent transition spaces）中进行强化学习来缓解这一问题 [10, 73, 74, 91]，在避免显式模拟器 rollout 的同时，仍然提升长时程推理能力。相比之下，我们的框架通过统一的 GRPO 目标同时优化决策选择和轨迹规划。通过在结构化的决策-规划层级（decision-planning hierarchy）上进行策略优化，我们的方法不仅在 closed-loop 指标下提升规划质量，还显式细化了决策空间本身，从而能够更有效地优化长时程驾驶行为。  
简要解释：这一段是在说明，传统后训练多半只改“怎么走”，而本文的方法同时改“先做什么决定”和“再怎么规划轨迹”，因此更适合端到端优化驾驶策略。

简要解释：总损失把分类和回归两类目标统一起来，使模型既能预测离散 token，又能优化连续运动量，从而覆盖视觉、动作和轨迹等不同学习信号。

**7.2.4 基准细节**

我们使用 Predictive Driver Model Score（PDMS）及其在 NAVSIM 中使用的扩展版本 EPDMS 来评估规划性能。PDMS 最早在 NAVSIM v1 [17] 中作为一种基于仿真的 open-loop 指标提出，其中将预测的 4 秒轨迹在 BEV 模拟器中展开，并通过将乘性安全约束与加权的规划质量子分数结合来打分。其定义为：

\[
\mathrm{PDMS}
=
\left(
\frac{\prod_{m\in\{\mathrm{TTC},\mathrm{EP},\mathrm{C}\}} w_m\, m(\mathrm{agent})}
{\prod_{m\in\{\mathrm{TTC},\mathrm{EP},\mathrm{C}\}} w_m}
\right)
\cdot
\prod_{m\in\{\mathrm{NC},\mathrm{DAC}\}} m(\mathrm{agent}) .
\tag{17}
\]

简要解释：这一段在说明 PDMS 的构成方式。它把安全性相关的乘性约束和规划质量相关的加权项结合起来，用于衡量自动驾驶规划输出的整体质量。

**评估指标。**

其中，NC 表示无责任碰撞（no at-fault collision），DAC 表示可行驶区域合规性（drivable-area compliance），TTC 表示碰撞时间（time-to-collision），EP 表示自车进度（ego progress），C 表示舒适性（comfort）。NAVSIM v2 [8] 进一步采用扩展的 Predictive Driver Model Score（EPDMS），在乘性评分中增加了行驶方向合规性（DDC）和信号灯合规性（TLC），并在加权评分中增加了车道保持（LK）、历史舒适性（HC）以及扩展舒适性（EC）。EPDMS 进一步加入如下过滤项：

\[
\mathrm{filter}_m(\mathrm{agent}, \mathrm{human}) =
\begin{cases}
1.0, & \text{if } m(\mathrm{human}) = 0,\\
m(\mathrm{agent}), & \text{otherwise}.
\end{cases}
\tag{18}
\]

这种过滤方式避免了当人类演示中也存在同样违规时对规划器进行惩罚；这种情况可能由于标注噪声或在特定上下文中合理的机动动作而发生。EPDMS 中加权项的权重为 \(w_{EP}=5\)、\(w_{TTC}=5\)、\(w_{LK}=2\)、\(w_{HC}=2\)、\(w_{EC}=2\)。

简要解释：这一段解释了 EPDMS 相比 PDMS 的扩展点，以及为什么要引入过滤机制。核心目的是避免把人类示范里本来就存在的“违规”错误算到模型头上，从而让评估更公平。

\]

\[
\mathbb{E}_Z\, TC(Y_U \mid C, Z) = TC(Y_U \mid C) - R_Z(U \mid C).

\[
\mathbb{E}_Z\, TC(Y_U \mid C, Z)
=
\sum_{i\in U} H(Y_i \mid C, Z) - H(Y_U \mid C, Z).
\tag{27}
\]

由于

\[
H(Y_i \mid C, Z) = H(Y_i \mid C) - I(Y_i; Z \mid C),
\tag{28}
\]

\[
H(Y_U \mid C, Z) = H(Y_U \mid C) - I(Y_U; Z \mid C),
\tag{29}
\]

\[
\mathbb{E}_Z\, TC(Y_U \mid C, Z)
=

### 7.3.3 残差混合下的正冗余增益（Positive Redundancy Gain under Residual Mixing）

我们现在给出足够条件，在这些条件下，一个潜在骨架（latent skeleton）会产生正冗余增益。

**假设 1：上游共同原因骨架（Upstream common-cause skeleton）。** 给定上下文 \(C\)，潜在骨架 \(Z\) 是一个上游的低频变量，它约束细粒度 token 组 \(Y_U\) 的生成：
\[
C \to Z \to Y_U .
\tag{31}
\]
这排除了在观察完整 token 组之后才计算得到的下游轨迹摘要、评估标签或选择变量。

**简要解释：** 这条假设要求 \(Z\) 必须是“先于”细粒度 token 生成的上游变量，而不是事后总结出来的结果变量；这样它才适合作为解释未来 token 结构的骨架。

**假设 2：骨架前依赖下界（Pre-skeleton dependence lower bound）。** 存在 \(\kappa(U) > 0\)，使得
\[
TC(Y_U \mid C) \ge \kappa(U).
\tag{32}
\]
这意味着在条件化于骨架之前，这个 token 组已经包含非平凡的组级依赖（group-level dependence）。

**简要解释：** 如果 token 之间本来就几乎没有共同结构，那么再引入骨架也很难带来额外收益；这里要求它们在进入骨架条件化前就存在足够的整体相关性。

**假设 3：骨架条件化后的残差混合（Residual mixing after skeleton conditioning）。** 在条件化于 \((C, Z)\) 之后，细粒度 token 之间的残余依赖随距离衰减：
\[
I(Y_i ; Y_j \mid C, Z) \le \beta \exp\!\left(-\frac{d(i, j)}{\ell_z}\right),
\tag{33}
\]
其中，\(\beta\) 衡量残余局部耦合强度，\(d(i,j)\) 是 token \(i\) 与 \(j\) 之间的距离，\(\ell_z\) 是残余相关长度（residual correlation length）。在该假设下，骨架条件化后的残差总相关（skeleton-conditioned residual total correlation）满足上界
\[
\mathbb{E}_Z\, TC(Y_U \mid C, Z) \le \epsilon_z(U),
\tag{34}
\]
其中
\[
\epsilon_z(U) = \mathbb{E}_Z \left[
\sum_{\{i,j\}\subset U}
\beta \exp\!\left(-\frac{d(i, j)}{\ell_z}\right)
\right].
\tag{35}
\]

**简要解释：** 这条假设表示：一旦已知骨架 \(Z\)，token 之间剩下的相关性应该主要是局部的、快速衰减的“残差项”，因此整体残差总相关可以用一个可计算的上界来控制。

结合潜在骨架分解与上述假设，有
\[
R_Z(U \mid C) = TC(Y_U \mid C) - \mathbb{E}_Z\, TC(Y_U \mid C, Z)
\ge \kappa(U) - \epsilon_z(U).
\]
因此，如果
\[
\kappa(U) > \epsilon_z(U),
\tag{37}
\]
则
\[
R_Z(U \mid C) > 0.
\tag{38}
\]

**简要解释：** 这说明只要骨架前的整体依赖强于骨架条件化后的残差依赖，那么骨架 \(Z\) 就带来正的冗余增益，即它确实帮助解释并压缩了 token 组中的共享信息。

等价地，在残差混合上界下，一个可验证的充分条件是
\[
\kappa(U) >
\mathbb{E}_Z \left[
\sum_{\{i,j\}\subset U}
\beta \exp\!\left(-\frac{d(i, j)}{\ell_z}\right)
\right]
\Longrightarrow
R_Z(U \mid C) > 0.
\tag{39}
\]

**简要解释：** 这个条件更直观：只要骨架前的依赖强度超过骨架后剩余的局部残差相关，就能保证正冗余增益成立。

这个条件对轨迹规划（trajectory planning）有直接解释。只有当骨架 \(Z\) 能解释多个未来 action token 共享的强低频依赖，并且在条件化之后只留下较弱的局部残余依赖时，它才是有用的。如果 \(Z\) 很难预测、充当一个下游碰撞点（downstream collider），或者不能减少残余依赖，那么正增益条件可能不成立。

**简要解释：** 这一段把理论条件落到规划任务上：好的骨架应该抓住多个未来动作 token 的共同低频结构；如果它只是“难预测的中间变量”或不能真正降依赖，就无法带来有效收益。
